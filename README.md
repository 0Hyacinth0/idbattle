# id大作战

这是一个纯前端的战斗模拟小工具。仓库提供了战斗逻辑、UI 以及用于回归的 Node 测试。下面是快速运行与开发指南，帮助你在本地复现设计稿中的页面效果。

## 环境准备

- Node.js 18 或更高版本（建议与测试用例保持一致）
- npm 9 以上

克隆项目后先安装依赖（本仓库目前没有额外依赖，但执行一次可以确保 npm 为后续脚本做好准备）：

```bash
npm install
```

## 启动开发服务器

项目是静态站点，只需要本地静态资源服务器即可。已经在 `package.json` 中提供了 `start` 命令，会启动一个无缓存的本地服务器，保证样式和脚本能够正确加载。

```bash
npm start
```

运行成功后命令行会输出访问地址（默认 `http://0.0.0.0:4173`，在浏览器中访问 `http://localhost:4173` 即可）。

如果你使用的是 VS Code，也可以直接右键 `index.html` 选择 “Open with Live Server”，效果与 `npm start` 一致。

## 运行测试

项目带有一组 Node Test，用于验证数值逻辑是否正确：

```bash
npm test
```

全部用例通过即可确认战斗核心逻辑无回归。

## 常见问题

- **页面样式不对或字体缺失**：确保是通过本地服务器访问页面，而不是直接双击打开 `index.html`。浏览器直接打开文件时，部分字体与跨域资源会被拦截，导致样式和设计稿不一致。
- **端口被占用**：可以在启动命令前临时指定 `PORT` 环境变量，例如 `PORT=3000 npm start`。

祝你调试顺利，玩得开心！
